SMS OTP Setup Guide

Prerequisites

  • SMS Gateway Provider account (Twilio, AWS SNS, Nexmo)
  • API credentials and authentication tokens
  • Phone number verification system
  • Database for storing user phone numbers

Step-by-Step Implementation

  1. Choose SMS Provider

    Select a reliable SMS gateway (Twilio recommended for global coverage)

  2. Configure API Credentials

    Set up API keys, account SID, and authentication tokens

  3. Implement OTP Generation

    Create secure random 6-digit code generator with expiration (5-10 minutes)

  4. Store OTP Securely

    Hash and store OTP in database with timestamp and user ID

  5. Send SMS

    Use provider API to send OTP to user's verified phone number

  6. Implement Verification

    Compare entered code with stored hash, check expiration

  7. Add Rate Limiting

    Limit attempts to 3-5 per hour to prevent abuse

Security Best Practices

  • Use HTTPS for all API communications
  • Implement rate limiting (max 3 SMS per 5 minutes)
  • Set OTP expiration to 5-10 minutes
  • Log all SMS attempts for audit trail
  • Validate phone numbers before sending
  • Use alphanumeric codes for higher entropy

Cost Considerations

Average cost: $0.01 - $0.05 per SMS. Budget accordingly for user base.

Email OTP Setup Guide

Prerequisites

  • SMTP server or email service (SendGrid, AWS SES, Mailgun)
  • Domain with SPF, DKIM, and DMARC configured
  • Email templates designed
  • User email verification system

Step-by-Step Implementation

  1. Configure Email Service

    Set up SMTP credentials or API keys for email provider

  2. Design Email Template

    Create branded HTML email with clear OTP display

  3. Implement SPF/DKIM/DMARC

    Configure DNS records to prevent emails going to spam

  4. Generate OTP Code

    Create 6-8 digit code with 10-15 minute expiration

  5. Send Email

    Use email service API to deliver OTP to verified email

  6. Track Delivery

    Monitor bounce rates and delivery status

  7. Verify Code

    Validate entered code against stored hash

Email Template Best Practices

  • Use clear subject line: "Your Verification Code"
  • Display OTP prominently in large font
  • Include expiration time clearly
  • Add security warning about not sharing code
  • Provide support contact information
  • Make it mobile-responsive

Deliverability Tips

Maintain sender reputation, use dedicated IP, monitor bounce rates, implement feedback loops.

App-based OTP (TOTP) Setup Guide

Prerequisites

  • TOTP library (pyotp, speakeasy, otplib)
  • QR code generation library
  • Secure secret key storage
  • Time synchronization (NTP)

Step-by-Step Implementation

  1. Generate Secret Key

    Create unique 32-character base32 secret for each user

  2. Store Secret Securely

    Encrypt and store secret in database, never expose to client

  3. Generate QR Code

    Create otpauth:// URI and convert to QR code for scanning

  4. Display Setup Instructions

    Show QR code and manual entry option to user

  5. Verify Initial Setup

    Ask user to enter first TOTP code to confirm setup

  6. Implement Verification

    Validate TOTP codes with time window tolerance (Â±30 seconds)

  7. Provide Backup Codes

    Generate 10 single-use backup codes for device loss

TOTP Configuration

  • Algorithm: HMAC-SHA1 (RFC 6238 standard)
  • Time step: 30 seconds
  • Code length: 6 digits
  • Time tolerance: Â±1 window (30 seconds)
  • Secret length: 160 bits (32 base32 characters)

Compatible Apps

Google Authenticator, Microsoft Authenticator, Authy, 1Password, LastPass Authenticator

Biometric Authentication Setup Guide

Prerequisites

  • FIDO2/WebAuthn compatible server library
  • HTTPS enabled (required for WebAuthn)
  • Compatible devices (Touch ID, Face ID, Windows Hello)
  • Fallback authentication method

Step-by-Step Implementation

  1. Implement WebAuthn API

    Use navigator.credentials.create() for registration

  2. Registration Flow

    Generate challenge, collect biometric data, store public key

  3. Store Credentials

    Save credential ID and public key in database

  4. Authentication Flow

    Generate challenge, verify signature with stored public key

  5. Handle Multiple Devices

    Allow users to register multiple biometric devices

  6. Implement Fallback

    Provide alternative authentication if biometric fails

  7. Privacy Compliance

    Ensure GDPR/CCPA compliance for biometric data

Security Considerations

  • Never store raw biometric data
  • Use device-local biometric verification
  • Implement attestation verification
  • Require user presence verification
  • Support multiple authenticators per user
  • Provide device management interface

Supported Platforms

iOS (Touch ID, Face ID), Android (Fingerprint, Face Unlock), Windows (Windows Hello), macOS (Touch ID)

Hardware Tokens Setup Guide

Prerequisites

  • FIDO2/U2F compatible server library
  • Hardware security keys (YubiKey, Titan, etc.)
  • HTTPS enabled website
  • Token management system

Step-by-Step Implementation

  1. Choose Token Standard

    Implement FIDO2/WebAuthn for modern security keys

  2. Procurement

    Purchase hardware tokens in bulk for organization

  3. Registration Process

    User inserts token, taps button, system stores public key

  4. Store Token Data

    Save credential ID, public key, and token metadata

  5. Authentication Flow

    Challenge-response with cryptographic verification

  6. Token Management

    Allow users to register multiple tokens, revoke lost ones

  7. Backup Strategy

    Require users to register 2+ tokens or backup method

Recommended Hardware

  • YubiKey 5 Series (USB-A, USB-C, NFC)
  • Google Titan Security Key
  • Feitian ePass FIDO
  • Thetis FIDO U2F Security Key
  • SoloKeys

Deployment Considerations

Budget $20-50 per token, plan for replacements, provide user training, establish token distribution process.

SSO Integration Setup Guide

Prerequisites

  • Identity Provider (IdP) - Okta, Auth0, Azure AD
  • SAML 2.0 or OAuth 2.0/OIDC library
  • SSL certificates
  • Metadata exchange capability

Step-by-Step Implementation

  1. Choose Protocol

    SAML 2.0 for enterprise, OAuth 2.0/OIDC for modern apps

  2. Configure Identity Provider

    Set up application in IdP, obtain metadata/credentials

  3. Exchange Metadata

    Import IdP metadata, export SP metadata to IdP

  4. Implement SSO Endpoints

    Create login, callback, and logout endpoints

  5. Add MFA to SSO Flow

    Configure MFA policies in IdP settings

  6. Test Authentication

    Verify login flow, attribute mapping, and MFA triggers

  7. Implement Session Management

    Handle session timeouts and single logout

MFA Integration Points

  • Configure MFA policies in IdP
  • Set up adaptive authentication rules
  • Define MFA requirements per application
  • Implement step-up authentication
  • Configure remember device options

Popular SSO Providers

Okta, Auth0, Azure AD, OneLogin, Ping Identity, Google Workspace, AWS SSO

VPN Integration Setup Guide

Prerequisites

  • VPN server (OpenVPN, WireGuard, Cisco AnyConnect)
  • RADIUS server for authentication
  • MFA provider with RADIUS support
  • Network infrastructure access

Step-by-Step Implementation

  1. Choose VPN Solution

    Select VPN that supports RADIUS authentication

  2. Set Up RADIUS Server

    Install FreeRADIUS or use cloud RADIUS service

  3. Configure MFA Provider

    Connect MFA service to RADIUS (Duo, Okta, etc.)

  4. Integrate VPN with RADIUS

    Configure VPN to use RADIUS for authentication

  5. Set Up MFA Policies

    Define which users/groups require MFA

  6. Test Connection

    Verify VPN login triggers MFA prompt

  7. Deploy to Users

    Distribute VPN client and setup instructions

VPN + MFA Flow

  • User enters username and password
  • VPN forwards to RADIUS server
  • RADIUS triggers MFA challenge
  • User completes MFA (push, OTP, etc.)
  • RADIUS returns success/failure
  • VPN grants or denies access

Recommended Solutions

Duo Security + OpenVPN, Okta + Cisco AnyConnect, Azure MFA + Azure VPN Gateway
